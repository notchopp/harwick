The Product
Not a chatbot. A front desk.
Most AI tools for real estate are lead forms with a chatbot strapped on. Harwick is different. It's a full intake-to-routing system — modeled as a typed AI runtime that holds conversation state, reads listing context, qualifies leads one question at a time, and knows when to step back for a human. Every decision is explainable. Every action is logged. The agent always knows why.
The core insight behind Harwick: real estate lead loss doesn't happen because agents are lazy. It happens because the gap between lead arrival and first qualified response is too slow, too manual, or too fragmented across channels. Harwick closes that gap entirely.

The Workflow
Lead arrival to agent handoff
Every lead follows the same path — regardless of channel. Harwick normalizes everything into one intake flow, one lead record, and one conversation thread.
Phase 1
Lead Arrives
A prospect DMs a listing post, comments on a reel, or calls a routed number. Harwick validates the payload, normalizes it into an internal lead event, and creates or updates the lead record. Post context — captions, listing hints, CTA — is hydrated automatically.
Instant capture · All channels
Phase 2
AI Holds the Conversation
Harwick AI generates a context-aware reply informed by the conversation history, listing inventory, workspace tone profile, and the post the lead came from. High-confidence replies auto-send. Anything uncertain surfaces in the operator queue for human approval before it goes out.
Context-aware · Tone-matched
Phase 3
Sequential Qualification
One question at a time — intent, timeline, price range, area, property type, financing status. Each answer updates the lead score and qualification state in real time. The AI never interrogates. It converses. Every signal feeds the routing decision downstream.
Score updates live
Phase 4
Human Takeover
Any operator or agent can pause AI on any individual thread and take over manually. The AI watches, retains context, and picks back up the moment the operator resumes. Takeover is per-conversation — other leads on the same team keep running on AI in parallel. Pause one, not all.
Thread-scoped · Full context preserved
Phase 5
Intelligent Routing
Once qualified, Harwick routes by area coverage, property type expertise, price range, and agent availability. The routing decision is always shown with its reasoning — not a black box. Operators can override. Everything is auditable in the lead timeline.
Explainable · Overrideable
Phase 6
FUB Handoff
Qualified lead + full conversation thread + qualification summary + suggested next action syncs to Follow Up Boss. Agent wakes up to a contact they can close — not one they need to re-qualify. Back-sync reconciliation prevents echo loops and keeps both systems accurate.
Full context preserved
The AI Runtime
How Harwick AI actually thinks
Harwick AI is not a prompt wrapper. It is a typed runtime — each turn returns a full conversation response with qualification state, tool calls, confidence scores, safety flags, and memory patches. The human only sees the reply. The system sees everything.
Conversation Memory
Isolated per thread
Each conversation has its own persistent state. When a lead resumes after 3 days, Harwick picks up exactly where it left off — buyer intent, price range, questions asked, questions not yet answered.
Automation Policy
AI decides. Policy executes.
The AI returns what should happen. The automation policy decides whether it's allowed to auto-send, queue for approval, or block entirely. Safe qualification replies auto-send. Showing requests and CRM sync stay approval-gated.
Tool Calls
Real actions, not just text
The runtime issues typed tool calls — send_meta_reply, route_lead, sync_follow_up_boss, pause_automation. Every action is governed, logged, and reversible.
Safety & Confidence
It knows when to stop
Every AI turn includes a confidence score and safety flags. Legal questions, lending topics, valuation estimates — these trigger automatic escalation to a human queue instead of an AI-generated response that could create liability.
Listing Context
Knows your inventory
Harwick reads from workspace listing facts — price, beds, baths, status, verification state — before every reply. It never guesses. If a listing is unverified or stale, it flags for operator review instead of hallucinating details.
Parallel Convos
Many threads. One team.
An agent can be manually working conversations 1 and 3 while Harwick runs conversations 2, 4, and 5 — all simultaneously, all isolated. No conversation bleeds into another. No context is shared across leads.
Architecture
Built for teams, not just solo agents
Harwick is multi-tenant and multi-user at its core. Each workspace is fully isolated — its own integrations, routing rules, listing inventory, agent profiles, and conversation history. A brokerage and a solo agent run on the same infrastructure, scoped completely separately.

Team Roles
Who does what inside Harwick
Agent
Receives qualified leads ready to act on. Can pause AI, take over a thread, and resume at any time without touching other conversations.
  • Work assigned lead queue
  • Pause / resume AI per conversation
  • Set routing profile (area, type, price range)
  • View full conversation history and qualification
  • Receive callback and showing tasks
Operator / ISA
Manages the shared work queue. Reviews AI drafts, approves or edits before send, handles edge cases the AI escalates.
  • Approve or dismiss AI reply drafts
  • Take manual control of any conversation
  • Reassign leads between agents
  • Manage unrouted and high-touch leads
  • Review routing decisions and overrides
Team Lead
Manages team routing rules, capacity, and oversight. Sees team-wide performance and can override any assignment or escalation.

  • View team leads queue and performance

  • Manage member profiles and availability
  • Access team-level activity log
Owner / Broker
Controls workspace integrations, billing, and full system visibility. Receives unrouted high-touch leads. Can override anything.

  • Manage plan and workspace settings
Design Principles
How Harwick is built to behave
Automation pauses for humans
AI handles routine qualification. Humans handle exceptions. The system always tells you why it paused — no silent failures, no mystery states. An operator can always see what the AI was doing and step in cleanly.
Context preserved everywhere
Every action — from first reply to final FUB sync — carries full context. No re-qualification. No "who was this person again?" The agent always arrives informed.
Routing is explainable
Why did this lead go to this agent? Show the work. Score breakdown. Match criteria. Area, type, capacity. Operators can always see and override the reasoning — the system is never a black box.
One lead, one thread
Instagram DM, Facebook comment, SMS, voice call — they're all one lead with one conversation history. No channel silos. No context jumps. Whether the lead calls back three days later or DMs from a different account, Harwick connects it.
The AI never guesses
If a listing is unverified, the AI does not fabricate details. If a lead asks a legal or lending question, it escalates instead of answering. Confidence scores gate every auto-send. When in doubt, it surfaces — it never wings it.
